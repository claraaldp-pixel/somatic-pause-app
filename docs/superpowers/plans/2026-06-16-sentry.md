# Sentry Error Monitoring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Sentry error monitoring to the Somatic Pause App — catching unhandled JS exceptions and React render errors in production, tagged with user ID, with readable stack traces via sourcemap upload.

**Architecture:** `Sentry.init()` runs before React mounts in `main.jsx`, wrapping `<App>` in `<Sentry.ErrorBoundary>`. User identity is tagged via `Sentry.setUser({ id })` inside `AuthContext.jsx`'s `checkAccess` function (set on auth, cleared on sign-out). Sourcemaps are generated at build time and uploaded to Sentry via `@sentry/vite-plugin`, then deleted from the public dist. Privacy controls disable console breadcrumbs and strip `event.extra` before events leave the browser.

**Tech Stack:** `@sentry/react`, `@sentry/vite-plugin`, Vite, Vitest, React 18

**Spec:** `docs/superpowers/specs/2026-06-16-sentry-design.md`

```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import * as Sentry from '@sentry/react'
import App from '@/App.jsx'
import '@/index.css'

Sentry.init({
  dsn: import.meta.env.VITE_SENTRY_DSN,
  environment: import.meta.env.MODE,
  sendDefaultPii: false,
  integrations: [Sentry.breadcrumbsIntegration({ console: false })],
  beforeSend(event) {
    delete event.extra;
    return event;
  },
});

ReactDOM.createRoot(document.getElementById('root')).render(
  <Sentry.ErrorBoundary fallback={<p>Something went wrong. Please refresh the page.</p>}>
    <App />
  </Sentry.ErrorBoundary>
)
```

```js
export default defineConfig({
  plugins: [
    react(),
    ...(process.env.SENTRY_AUTH_TOKEN ? [

        sourcemaps: { filesToDeleteAfterUpload: ['./dist/**/*.map'] },

  resolve: {
```

## Task 5: Configure Vercel environment variables

**This task is manual setup in the Vercel dashboard — no code changes.**

You will need four values from your Sentry account before starting:
- **DSN**: Sentry project → Settings → Client Keys (DSN)
- **Auth token**: Sentry account → Settings → Auth Tokens → Create new token (scope: `project:releases` + `org:read`)
- **Org slug**: visible in Sentry URL: `sentry.io/organizations/<org-slug>/`
- **Project slug**: Sentry project → Settings → General → Project slug

- [ ] **Step 1: Add environment variables in Vercel**

Go to Vercel → somatic-pause-app → Settings → Environment Variables. Add all four for the **Production** environment:

| Variable | Value | Secret |
|---|---|---|
| `VITE_SENTRY_DSN` | your DSN from Sentry | No |
| `SENTRY_AUTH_TOKEN` | your auth token | Yes (mark as sensitive) |
| `SENTRY_ORG` | your org slug | No |
| `SENTRY_PROJECT` | your project slug | No |
